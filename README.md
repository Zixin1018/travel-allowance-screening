# 差补重复筛查工具 v3

从月度差补核对报表自动筛查两类问题：单流程人工计算错误多发、多流程日期重叠重复发放。

联动 `境外差旅补贴计算器` 自动识别境外流程，按对应国家标准 + 返程日汇率换算比对。

## 用法

```bash
python3 差补筛查.py <报表文件.xlsx>
```

输出文件生成在输入文件同目录下：`差补筛查结果-{YYYY}年{MM}月.xlsx`。

---

## 输入要求

报表需含两个 Sheet，其他 Sheet 忽略：

### 主表数据（列位置固定）

| 列 | 字段 | 用途 |
|----|------|------|
| A | 流程编号 | 主键，格式 `出差申请202605639752` |
| C | 工号 | 员工标识 |
| D | 出差人 | 姓名 |
| J | 起止时间 | 格式 `2026-05-05~2026-05-15`，解析为起/止日期 |
| L | 差补 | 实发差补总额（元） |

> 不依赖用户手动添加的「明细补助匹配」「差额」列，这些列可有可无。

### 明细表数据（列位置固定）

| 列 | 字段 | 用途 |
|----|------|------|
| A | 流程编号 | 关联主表 |
| C | 工号 | 备用（仅明细有的流程从这里取） |
| D | 出差人 | 备用 |
| H | 起止时间 | 该段行程的起止日期 |
| I | 出发地 | 如 `广东-深圳`、`亚洲-香港` |
| J | 目的地 | 如 `亚洲-香港`、`欧洲-俄罗斯-圣彼得堡` |
| Q | 伙食补助 | 该段行程实发伙食（元） |
| R | 交通补助 | 该段行程实发交通（元） |
| S | 差补说明 | 备注（聚合到原始数据 Sheet） |

> 不依赖用户手动添加的「伙食+交通补助」「起始时间」「结束时间」列。

### 自动过滤

- 伙食补助和交通补助均为 0 的明细行跳过
- 主表差补 = 0 且明细汇总 = 0 的流程跳过
- 明细表有但主表无的流程，只要金额 > 0 就**补充纳入**（从明细取工号和姓名）

## 数据处理流程

```
主表数据 (1804条)          明细表数据 (4141条)
      │                          │
      │  ┌───────────────────────┘
      │  │ GROUP BY 流程编号
      │  │  SUM(伙食补助) SUM(交通补助)
      │  │  最早起始日 最晚结束日
      │  │  收集出发地/目的地 → 判断境外
      │  │  收集差补说明
      │  ▼
      ├──● LEFT JOIN（主表为主，明细补充）
      │
      ▼
  差补重复分析 (1865条)
      │
      ├──→ 境内 → 100/80标准筛查
      │         ├─ 单流程金额检查
      │         └─ 重叠重复发放检查
      │
      └──→ 境外 → 联动境外计算器
                ├─ 匹配273条城市标准
                ├─ 取返程日汇率换算
                └─ 比对实发 vs 应有
```

---

## 日期解析

「起止时间」字段统一格式为 `YYYY-MM-DD~YYYY-MM-DD`（如 `2026-05-05~2026-05-15`）。

天数计算：`(结束日 - 起始日).days + 1`，即首尾都算。例如 5/5~5/15 = 11 天。

---

## 境内筛查规则

### 标准

境内统一使用：
- 伙食补助 100 元/天
- 交通补助 80 元/天

### 单流程金额检查

```
应有伙食 = 天数 × 100
应有交通 = 天数 × 80
实发 > 应有 → 标记「单流程金额偏大」
实发 ≤ 应有 → 正常（少发说明会计核减，不管）
```

### 重叠重复发放检查

同一员工的所有流程（境内+境外）统一处理：

1. 按天展开所有流程的覆盖日期
2. **同一天境外和境内都覆盖 → 算境外天**（境外优先，补助更高）
3. 同一天仅境内多流程覆盖 → 合并去重算 1 天
4. 境外天 × 境外标准 × 汇率 + 境内天 × 100/80 = 应有总额
5. 所有流程实发合计 > 应有总额 → 标记「重复发放」

## 境外筛查规则（联动境外计算器）

### 境外判定

明细表中**任一**行程的目的地以以下前缀开头，整个流程判为境外：

- `亚洲-` `欧洲-` `美洲-` `非洲-` `大洋洲-`

示例：
- `亚洲-香港` → 境外，国家 = 香港
- `欧洲-俄罗斯-圣彼得堡` → 境外，国家 = 俄罗斯，城市 = 圣彼得堡
- `广东-深圳` → 境内

### 标准匹配

从 `差旅补贴计算.html` 中提取 273 条城市标准，每条结构：

```json
{"country": "香港", "city": "", "currency": "HKD", "food": 500, "public": 300}
```

匹配顺序：
1. 解析目的地 → 国家 + 城市
2. 在国家下找匹配城市（如「圣彼得堡」匹配「叶卡捷琳堡、圣彼得堡」）
3. 无城市匹配 → 用「其他城市」条目
4. 无其他城市 → 用该国第一条

### 汇率选取

**按返程日期（流程结束日期）取汇率**，整段行程统一用一个汇率。

汇率数据从 `差旅补贴计算.html` 的嵌入式 JSON 提取，覆盖 112 天，5 种币种（USD/EUR/JPY/HKD/GBP）。

返程日无汇率时（周末/节假日）→ 前后各找 30 天，取最近的有效汇率。30 天内都无 → 标记「返程日期无汇率」。

### 境外应有金额计算

```
应有伙食 = 伙食标准(原币/天) × 天数 × 返程日汇率
应有交通 = 公杂标准(原币/天) × 天数 × 返程日汇率
```

### 境外预警

| 预警 | 触发条件 |
|------|---------|
| 伙食多发 | 伙食差异 > 50 元 |
| 交通多发 | 交通差异 > 50 元 |
| 境外流程日期重叠 | 同一员工多个境外流程日期重叠 |
| 涉及多国 | 流程有多个境外目的地，仅用首国标准 |
| 返程日期无汇率 | 返程日 ±30 天内无汇率 |

> 少发（实发 < 应有）**不预警**——通常是会计根据实际行程手动核减了某些天，属于正常操作。

### 多国行程

一个流程涉及多个境外国家时（如庞志远：土耳其+俄罗斯），按字母序取首国标准计算，并标注「涉及多国(X,Y)，仅用首国标准计算」。此类流程建议人工复核。

## 输出文件结构

| Sheet | 行数 | 内容 |
|-------|------|------|
| 差补筛查明细 | 仅异常行 | 单流程偏大 + 重复发放，整行标黄，含应有/实发/差异/说明 |
| 原始数据(标黄) | 全量 | 每流程一行，含境内外、出发地、目的地、差补说明、境外标准，异常行整行标黄 |
| 汇总统计 | 全量员工 | 境外天/境内天分列，多发金额从高到低，多发红字 |
| 境外流程清单 | 仅境外 | 国家、币种、标准、返程汇率、应有/实发/差异、预警 |
| 原始主表数据 | 原表全量 | 原始主表 + 最末列「境内外」标记 |
| 原始明细表数据 | 原表全量 | 原始明细表 + 最末列「境内外」标记 |

---

## 跨月处理

筛查逻辑基于日期而非月份，5 月和 6 月数据混在一起不影响结果。输出文件名取数据中**日期的众数月份**。
